# `msg/` — Message Catalog Module

The `msg/` directory holds all human-readable strings for the CUBRID engine and its bundled tools, organized by locale. At runtime the engine calls POSIX `catgets` (via the compiled `.cat` binary) to look up a message by set ID and message number, format it with `printf`-style positional arguments, and surface it through `er_msg()`.

For full format details and the C-side loader API, see [[components/message-catalog|message-catalog]].

## Locales and Files

| Source locale | Encoding | Notes |
|---|---|---|
| `en_US.utf8/` | UTF-8 | Canonical source; always authoritative |
| `en_US/` | UTF-8 | Build-time copy of `en_US.utf8/` (no transformation) |
| `ko_KR.utf8/` | UTF-8 | Korean translations (UTF-8 source) |
| `ko_KR.euckr/` | EUC-KR | Build-time `iconv` conversion of `ko_KR.utf8/` |

**Four locales are installed; only two are maintained by hand.** `en_US/` and `ko_KR.euckr/` are generated by the build system from their UTF-8 counterparts and are never edited directly.

## Catalog Files (per locale)

Three `.msg` source files exist in each canonical locale directory:

| File | POSIX set range | Covers |
|---|---|---|
| `cubrid.msg` | set 1 (GENERAL), set 4 (PARAMETER), set 5 (ERROR), … | Engine error messages (~1700 codes), system parameter diagnostics, generic argument parsing |
| `csql.msg` | set 1 (MSGCAT_CSQL_SET_CSQL) | CSQL interactive shell UI strings, option help text |
| `utils.msg` | sets 1–59 | Every admin utility: `createdb`, `backupdb`, `restoredb`, `loaddb`, `unloaddb`, `checkdb`, `diagdb`, `killtran`, `lockdb`, `tranlist`, `spacedb`, `compactdb`, `plandump`, `copylogdb`, `applylogdb`, `tde`, `flashback`, `memmon`, and ~35 others |

Each file is compiled to a `.cat` binary by `gencat` at build time.

## Build Pipeline (`msg/CMakeLists.txt`)

```
CMake target: gen_msgs_<locale>
  For each locale × {cubrid, csql, utils}:
    1. [en_US]     cmake -E copy    en_US.utf8/X.msg  → en_US/X.msg
    2. [ko_KR.euckr] iconv -f utf-8 -t euckr ko_KR.utf8/X.msg → ko_KR.euckr/X.msg
                   (Windows: powershell UTF8 → EUC-KR via .NET Encoding)
    3. gencat X.cat X.msg
  install: both .msg and .cat go to ${CUBRID_LOCALEDIR}/<locale>/
```

The `iconv` command is a hard dependency on non-Windows; CMake will abort (`FATAL_ERROR`) if it is missing.

## Connection to the 6-Place Error-Code Rule

> [!warning] Six-place update rule
> Adding a new CUBRID error code requires simultaneous edits in six places. `msg/en_US.utf8/cubrid.msg` (place 3) and `msg/ko_KR.utf8/cubrid.msg` (place 4) are among them. Omitting either locale causes `catgets` to fall back to the generic "Missing message for error code %d" string at runtime and breaks Korean users silently.
>
> Full rule: [[Error Handling Convention]] | [[components/error-manager|error-manager]]

## Message Format (POSIX catgets)

Messages use POSIX message-catalog syntax:

```
$set <N> <SYMBOLIC_NAME>
<msg-num> format string with %N$<type> positional args
```

Example from `en_US.utf8/cubrid.msg`:

```
$set 5 MSGCAT_SET_ERROR
3 Out of virtual memory: unable to allocate %1$zu memory bytes.
11 Unable to mount disk volume "%1$s". The database "%2$s", ...
```

Arguments are **positional** (`%1$s`, `%2$d`, `%3$lld`) — not sequential — which is what allows Korean translations to reorder subjects and objects without touching the C call sites.

## C-Side Lookup

At runtime `er_set()` (in [[components/error-manager|error-manager]]) calls `msgcat_message(MSGCAT_CATALOG_CUBRID, MSGCAT_SET_ERROR, error_code_ordinal)` to retrieve the format string, then `vsprintf`-formats it with the variadic arguments. See [[components/message-catalog|message-catalog]] for the full loader lifecycle.
