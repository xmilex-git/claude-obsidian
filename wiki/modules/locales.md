# `locales/` ‚Äî Locale Data + Build Toolchain

Source data and build scripts that produce CUBRID's per-locale shared libraries (collation tables, character set conversion, calendar/number/date format rules).

## Layout

| Subdir | Purpose |
|--------|---------|
| `data/ldml/` | Per-locale LDML (Unicode CLDR-style) XML files: `cubrid_de_DE.xml`, `cubrid_fr_FR.xml`, `cubrid_it_IT.xml`, `cubrid_es_ES.xml`, `cubrid_ja_JP.xml`, `cubrid_ko_KR.xml`, `cubrid_km_KH.xml`, `cubrid_ro_RO.xml`, `cubrid_tr_TR.xml`, `cubrid_vi_VN.xml`, `cubrid_zh_CN.xml` + `common_collations.xml` |
| `data/codepages/` | Single-byte and East-Asian codepage tables (8859-1/9/15, CP1250/1258, CP932/936/949 ‚Ä¶) |
| `data/ducet.txt` | Default Unicode Collation Element Table |
| `data/unicodedata.txt` | UnicodeData snapshot (general category, decomposition, etc.) |
| `loclib/` | `build_locale.sh` ‚Äî Linux build script |
| `loclib_win_Win32/`, `loclib_win_x64/` | Windows MSBuild project skeletons |
| `make_locale.sh`, `make_locale_*.bat` | Top-level driver scripts |

## Supported locales (out of the box)

`de_DE`, `es_ES`, `fr_FR`, `it_IT`, `ja_JP`, `ko_KR`, `km_KH`, `ro_RO`, `tr_TR`, `vi_VN`, `zh_CN` ‚Äî plus `en_US` baseline always available.

## Build pipeline

```
data/ldml/cubrid_<lang>.xml
data/codepages/*.TXT
data/ducet.txt
data/unicodedata.txt
        ‚îÇ
        ‚îÇ  make_locale.sh [-t 64|x86_64] [-m release|debug] [LOCALE]
        ‚îÇ  ‚Üí loclib/build_locale.sh
        ‚îÇ
        ‚ñº
   per-locale C source generated by `gen_locale` engine utility
        ‚îÇ
        ‚îÇ  gcc / cl.exe
        ‚ñº
   libcubrid_<lang>.so / .dll
        ‚îÇ
        ‚ñº
   installed under ${CUBRID_SHLOCALESDIR}, loaded by server at boot
```

`make_locale.sh` flags:
- `-t arg` ‚Äî target machine (`x86_64`, `x64`, `64`, `64bit`); default `64`
- `-m arg` ‚Äî build mode (`release` or `debug`); default `release`
- `LOCALE` ‚Äî optional name like `de_DE`; omit to build all configured locales

## Engine consumption

- The engine loads the per-locale `.so/.dll` at server boot ([[components/server-boot|server-boot]]).
- Active locale is selected per session via system parameter `intl_collation` and per-column `COLLATE` clauses (parsed in [[components/parser|parser]]).
- Collation rules drive `<`, `=`, ORDER BY, GROUP BY, and B-tree key comparison ([[components/btree|btree]]).
